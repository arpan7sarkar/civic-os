# PRD: Admin/Authority Dashboard Enhancement

## 1. Objective
To upgrade the existing Authority Dashboard into a comprehensive, professional, and visually consistent "Operational Control Center" that mirrors the high-quality design of the Citizen Dashboard while incorporating official government-style elements and deeper management capabilities.

## 2. Design Goals
- **Consistency:** Use the same `gov-blue` color palette, typography (System UI / Inter), and layout patterns as the Citizen Dashboard.
- **Official Aesthetics:** Incorporate government-style branding (Emblems, Departmental Headers, Official Seals, "DPDP Compliant" badges).
- **Professionalism:** Maintain a clean, high-contrast, and minimalist UI that feels authoritative and modern.

## 3. Key Enhancements

### 3.1. Visual & Branding`
- **Unified Sidebar:** Switch from the dark sidebar to a light, clean sidebar matching the Citizen Dashboard, but with admin-specific menus.
- **Government Header:** include the "Govt. of India" and "CivicOS Authority Portal" branding with the official logo.
- **Official Stamp/Seal:** Add a visual "Digital Seal" to resolved reports to simulate an official stamp of approval.
- **Typography:** Ensure 100% consistency with the citizen portal's font weights and tracking.

### 3.2. Operational Depth
- **Enhanced Statistic Cards:** Add trending indicators and "SLA Velocity" metrics.
- **Ward Analytics:** (Future) A dedicated view to see which wards are underperforming.
- **Resolution Timeline:** A detailed audit trail for each grievance (Who assigned it, when status changed, who uploaded the proof).
- **Official Notifications:** Alert system for overdue SLAs and critical priority issues.

### 3.3. Government-Specific Features
- **Circulars & Notices:** A sidebar section for internal government circulars or department-wide announcements.
- **Departmental Toggle:** Allow high-level admins to switch views between different departments (e.g., Sanitation, Roadway, Electricity).

## 4. UI/UX Requirements
- **Responsive Management:** The table/list view must be mobile-friendly for field officers.
- **Action Context:** Status updates must require clear proof (already implemented, but can be visually polished).
- **Search & Filter:** Advanced filtering by Ward, Priority, and SLA Status.

## 5. Success Metrics
- **Visual Parity:** Zero perceived difference in design quality between Citizen and Admin portals.
- **Information Density:** Increased data visibility for authorities without cluttering the UX.
- **Official Feel:** Subjective user feedback that the portal feels "Official" and "Trusted."
